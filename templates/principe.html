Principe de fonctionnement

Gmail → Lecture IMAP → Règles → Resend → Destinataires
1

RÉCEPTION DE L'EMAIL

Un email arrive dans la boîte Gmail configurée. Le système surveille cette boîte en permanence, en se connectant toutes les 30 secondes via le protocole IMAP (Internet Message Access Protocol), qui permet de lire les emails sans les télécharger définitivement.

2

DÉTECTION DES EMAILS NON LUS

À chaque cycle, l'application récupère uniquement les emails marqués comme "non lus" (UNSEEN), afin d'éviter de traiter plusieurs fois le même message. Une fois lu, l'email ne sera plus retraité lors des cycles suivants.

3

APPLICATION DES RÈGLES

Pour chaque email, le système vérifie d'abord si un mot-clé défini dans les règles apparaît dans le sujet ou le corps du message (priorité haute). Si aucun mot-clé ne correspond, il vérifie ensuite si l'expéditeur est reconnu. Si aucune règle ne correspond, l'email n'est pas transféré et un message est affiché dans le terminal.

4

TRANSFERT VIA RESEND

Si une règle correspond, l'email est retransmis aux destinataires définis, via l'API de Resend, depuis l'adresse contingence@projet.courses. Le sujet original est conservé, avec le préfixe [Transféré] ajouté automatiquement.

5

ENREGISTREMENT DANS LE JOURNAL

Chaque transfert réussi est enregistré dans une base de données locale (SQLite), avec la date, l'expéditeur, le sujet et les destinataires. Ces informations sont consultables à tout moment sur la page Journal du tableau de bord.

6

TABLEAU DE BORD WEB

Une interface web (ce tableau de bord) permet de consulter l'historique des transferts, de visualiser les règles actives, et de comprendre le fonctionnement du système, sans avoir besoin d'accéder au code source ou au terminal.

Note : Les deux composants de l'application (le programme de transfert app.py et le tableau de bord web web.py) fonctionnent de manière indépendante et doivent être lancés séparément dans deux terminaux distincts.
